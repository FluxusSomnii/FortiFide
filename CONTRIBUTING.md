# Contributing to Forti Fide

Forti Fide is open source under GPL v3.

## How to contribute

- **Bug reports** — Open an issue with description and reproduction steps
- **Pattern library** — Propose new rhetorical patterns with neutral definitions and confidence calibration notes
- **Bug fixes** — Submit a pull request referencing the issue
- **Documentation** — Always welcome

## The core constraint

The pattern detection methodology must remain transparent and auditable. No contribution may introduce hidden detection logic, undisclosed weighting, or any mechanism that tunes what gets flagged toward an undisclosed interest. The glass box is the product.

## Pattern library contributions

New patterns require:
1. A neutral name (descriptive, not pejorative)
2. A plain-language definition that cannot be gamed
3. A confidence calibration note
4. Example phrases
5. The register it belongs to (Fear / Identity / Authority / Intimacy / Rational)

## Pyannote licence note

The pyannote diarization model requires users to accept the licence on Hugging Face themselves. It cannot be bundled in the distribution. README installation instructions must document this explicitly.

## Contributor rewards

Meaningful contributions are eligible for rewards from the transparent donation pool. The contribution record is public.

## Code of conduct

The instrument exists to return awareness. Keep that as the north star.
